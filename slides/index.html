The Challenge

Why generic LLMs fall short

  • Role mismatch — NOC engineers and retail staff need different tone, tools, and escalation paths
  • Stale knowledge — plans, promotions, and support flows change constantly
  • Hallucination risk — billing and network guidance must be grounded in approved content
  • Demo constraints — enterprise buyers often require air-gapped or offline proof points

Goal: Ship a repeatable demo where each telco persona searches carrier-specific knowledge in Elasticsearch, answers through Elastic Agent Builder, and runs in the field without live cloud dependencies.

Knowledge Ingest Pipeline

Crawl — ingest/crawl.py

  • BFS over t-mobile.com from seed URLs
  • Jina Reader API → clean markdown + link extraction
  • Persona tags from URL heuristics (/support, /business, …)

Chunk — ingest/chunk.py

  • Heading-aware splits ~1000 chars with overlap
  • content + body_semantic fields

Index — ingest/index.py

PUT /_inference/text_embedding/
    jina-embeddings-v3

Index: telco-tmobile-kb
  body_semantic: semantic_text
  inference_id: jina-embeddings-v3

T-Mobile Persona Agents

Each agent shares one KB search tool but uses distinct instructions, tone, and escalation behavior.

NOC Engineer

tmobile-noc-engineer

Outages, 5G/core, coverage maps, business circuit escalation

Customer Care

tmobile-customer-care

Plans, devices, troubleshooting, support article guidance

Billing Ops

tmobile-billing-ops

Invoice lines, disputes, payment failures, proration

Retail Sales

tmobile-retail-sales

Promotions, upgrades, trade-in, in-store offer eligibility

Grounding rule (all personas): search the KB first, cite source URLs, never invent policy or pricing.

Live Demo Script

  1. Set context — "Same KB, four roles — watch tone and focus change."
  2. Customer Care — "What unlimited plans does T-Mobile offer?" → cites KB URLs
  3. NOC — "How do I verify 5G coverage for a business address?"
  4. Billing — "Where can customers see line-level charges?"
  5. Retail — "What should I highlight for a family plan upgrade?"
  6. Switch to offline — show local ES + RAG without Kibana connectivity

If an answer is thin, say: "KB crawl date matters — rerun bootstrap to refresh public content."
